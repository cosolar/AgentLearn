# 8.5 低代码/可视化 Agent 平台

## 📖 本章目标

- 了解主流低代码 Agent 平台的功能定位
- 掌握 Dify 搭建 Agent 应用的全流程
- 学会在编码方案与低代码方案间做选择

---

## 为什么需要低代码平台？

本教程前 7 章全是手写代码构建 Agent。但在实际项目中，**不是所有人都需要/适合写代码**：

```
纯编码方案（本教程主路线）：
✅ 最大灵活性
✅ 精细控制
✅ 易于集成 CI/CD
❌ 开发周期长
❌ 需要专业开发者

低代码/可视化方案：
✅ 快速原型
✅ 业务人员可用
✅ 内置最佳实践
❌ 灵活性受限
❌ 扩展困难
```

> 💡 **最佳实践**：低代码平台做快速验证，编码方案做深度定制，两者互补。

## Dify 深度解析

Dify 是目前最成熟的低代码 Agent 平台，本质上是**将本教程学到的一切（Agent、RAG、工具、工作流）做成了可视化界面**。

### Dify vs 本教程对照

| 本教程概念 | Dify 中的对应 |
|-----------|-------------|
| Prompt 工程 (2.1) | 提示词编辑器 |
| Chain 模式 (2.2) | 工作流节点 |
| Agent 架构 (2.3) | Agent 节点配置 |
| 记忆 (2.4) | 变量/对话历史 |
| 工具调用 (3.2) | 工具插件市场 |
| RAG 系统 (5章) | 知识库模块 |
| 多 Agent (6.1) | 工作流多 Agent 编排 |
| API 部署 (7.1) | 一键 API 发布 |

### Dify 核心功能

#### 1. 可视化工作流编辑器

```
用户输入
    │
    ▼
┌─────────────────┐
│  问题分类节点     │ ← LLM 判断问题类型
└────────┬────────┘
         │
    ┌────┴────┐
    ▼         ▼
┌────────┐ ┌────────┐
│ 知识库  │ │ 工具调用 │
│ 检索    │ │ (搜索)  │
└────┬───┘ └────┬───┘
    │         │
    └────┬────┘
         ▼
┌─────────────────┐
│  答案生成        │
└─────────────────┘
    │
    ▼
  输出
```

#### 2. 知识库管理

```yaml
支持格式: PDF, DOCX, TXT, Markdown, HTML, JSON, CSV, Excel
分块策略: 自动/自定义 (与 5.2 章同原理)
检索方式: 向量检索 / 全文检索 / 混合检索
嵌入模型: OpenAI / HuggingFace / 本地部署
```

#### 3. 工具插件生态

Dify 内置 50+ 工具插件，也支持自定义工具（你可以在本教程 3.2 章学会如何实现）：

```
搜索类: Google, Bing, DuckDuckGo, Tavily
数据处理: Excel, SQL, 代码执行器
通信类: Email, Slack, Discord, Telegram
图像类: DALL-E, Stable Diffusion
浏览器: Web Scraper, Jina AI
```

## Flowise — LangChain 可视化

Flowise 本质上是 **LangChain 的可视化封装**。如果你已经学会 LangChain（本教程第 3 章），会感觉非常亲切。

### Flowise 核心概念

```
拖拽组件 ↔ 对应 LangChain 组件
─────────────────────────────────
LLM Chain     ↔  prompt | llm | parser
Agent         ↔  AgentExecutor
Tool          ↔  @tool decorated function
Vector Store  ↔  VectorStoreRetriever
Memory        ↔  ChatMessageHistory
```

**适用场景：**
- 快速验证 LangChain 的编排思路
- 非开发者搭建简单链式应用
- 教学演示

---

## RAGFlow — 深度文档 RAG

RAGFlow 专注于**文档理解**，比通用平台在 RAG 场景上做得更深。

**核心差异：**
```
通用 RAG（本教程5章）：
  文档 → 分块 → 向量化 → 检索 → 生成

RAGFlow 增强：
  文档 → 布局分析 → OCR(图片) → 
  表格识别 → 分块(语义) → 
  混合检索(向量+关键词+重排序) → 生成
```

**适用场景：** 企业知识库、研究报告分析、合同审查、法律文档

## 选型建议

| 需求 | 推荐平台 | 理由 |
|------|---------|------|
| 快速搭建客服机器人 | Dify | 最全面的开箱即用 |
| 企业知识库 | RAGFlow | 深度文档理解 |
| LangChain 原型验证 | Flowise | 与编码方案无缝切换 |
| 个人文档助手 | AnythingLLM | 极简，本地优先 |
| 快速演示/MVP | AgentGPT | 无需部署 |
| 企业级合规知识库 | Cognita | MLOps + 权限控制 |

## 决策框架

```
你的目标是？
│
├── 快速验证想法 → 低代码平台 (Dify/Flowise)
│
├── 生产级应用？
│   ├── 业务逻辑简单 → Dify
│   ├── 文档理解为主 → RAGFlow
│   └── 逻辑复杂/需要定制 → 编码方案 (本教程)
│
├── 团队技能？
│   ├── 非技术团队 → 低代码平台
│   └── 开发团队 → 编码方案 + Dify 辅助
│
└── 长期维护？
    ├── 快速迭代 → 低代码平台
    └── 深度定制 → 编码方案
```
